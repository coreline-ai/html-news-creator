## 개발 목적
- OpenAI 일부 공식 페이지가 Cloudflare challenge로 `403 Forbidden`을 반환해 본문 추출이 실패하는 문제를 품질 관점에서 완화한다.
- 추출 실패 항목이 매 실행마다 반복 재시도되는 상황을 줄인다.
- 공식 RSS/수집 요약이 있는 항목은 `ExtractedContent`에 fallback 결과로 저장해 이후 단계가 일관된 데이터 경로를 사용하게 한다.

## 개발 범위
- `scripts/run_daily.py`의 extract 단계에서 `ExtractorOrchestrator`를 실제로 사용한다.
- 모든 extractor 실패 시 `RawItem.raw_text` 기반 fallback `ExtractResult`를 생성한다.
- fallback/실패 원인을 `ExtractedContent.metadata_json`에 기록한다.
- OpenAI 403 재발 방지와 fallback helper 동작을 단위 테스트로 검증한다.
- README에 403/RSS fallback 운영 원칙을 간단히 반영한다.
