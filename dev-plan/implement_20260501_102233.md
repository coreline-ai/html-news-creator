## 개발 목적
진행 중인 파이프라인 Run이 페이지 새로고침·탭 전환·다른 라우트 이동 후에도 추적되도록 `안 ①-R`을 구현한다. 핵심은 브라우저 localStorage에 `activeRun`을 저장하고, 앱 전체에서 단 하나의 SSE 구독자가 Run 진행 상태를 표시하도록 하는 것이다.

## 개발 범위
- Backend에 Run 상태 조회 API 추가: `GET /api/runs`, `GET /api/runs/{run_id}`.
- SSE late subscriber가 완료/실패 Run에 재접속해도 `done` 이벤트를 받을 수 있게 보강.
- `useAppStore`에 persisted `activeRun` slice 추가.
- `NewReport`는 Run 시작 후 `activeRun`만 저장하고 직접 SSE 구독하지 않도록 변경.
- `GlobalRunIndicator` 컴포넌트를 앱 루트에 장착해 전역 단일 SSE 구독과 우하단 진행 토스트를 담당.
- stale run(서버 재시작 등으로 404) 처리 및 안내 후 정리.
- 관련 단위 테스트와 E2E mock 보강.

## 제외 범위
- DB `JobRun`/`JobLog` 영속화 구현.
- multi-worker 안정성 보장.
- 서버 프로세스 재시작 후 subprocess 재연결.
- 신규 마이그레이션.
- 실행 로그 전체 UI/상세 로그 뷰어.
