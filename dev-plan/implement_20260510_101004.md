# implement_20260510_101004.md

작성 일시: `2026-05-10 10:10:04 KST`

이 문서는 이번 개발의 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적
달력에서 아직 리포트가 생성되지 않은 날짜를 클릭했을 때 `Could not load report / API 404` 오류 화면으로 보내지 않고, 사용자의 의도에 맞게 “해당 날짜 리포트 생성” 흐름으로 자연스럽게 연결한다. 이미 만들어진 날짜는 기존처럼 생성된 내용을 보여준다.

## 개발 범위
- Dashboard 달력 월별/히트맵 날짜 클릭 라우팅 분기
- `/reports/:date` 직접 진입 시 404를 “아직 생성되지 않음” 상태로 처리
- `/reports/new?date=YYYY-MM-DD` 진입 시 New Report의 Date 입력값 자동 프리셋
- 기존 생성 완료/초안/실패 리포트 열람 흐름 유지
- 위 분기별 단위 테스트 및 브라우저 스모크 테스트

## 제외 범위

## 문서 기반 제약 사항
- `docs/design/**` 디자인 시스템은 웹앱 UI용이며 정적 HTML 리포트 템플릿에 자동 적용하지 않는다.
- 웹앱은 React SPA + FastAPI API 구조이며, 실행은 `/api/runs`를 통해 subprocess로 처리한다.
- 기존 `ReviewReport`의 검토/발행 기능과 `GlobalRunIndicator` 완료 후 `/reports/{date}` 이동 흐름을 깨지 않는다.

## 케이스 구분 제안

| 케이스 | 현재 동작 | 목표 동작 | 권장 라우팅/화면 |
|---|---|---|---|
| A. 생성된 리포트 있음: `published`/`ready` | `/reports/{date}` 열람 | 유지 | `/reports/{date}`에서 HTML/섹션 표시 |
| B. 초안/부분 생성 리포트 있음: `draft` 등 | `/reports/{date}` 열람 | 유지하되 “초안/부분” 상태 명확화 | `/reports/{date}` 검토 화면 |
| C. 실패 이력 리포트 있음: `failed`/`error` | 달력 점 표시 후 상세 진입 | 실패 상세 또는 재생성 CTA 표시 | `/reports/{date}`에서 실패 상태 + “다시 생성” CTA |
| D. 리포트 없음: 달력에서 클릭 | `/reports/{date}` → 404 오류 | 생성 화면으로 이동 | `/reports/new?date={date}` |
| E. 리포트 없음: URL 직접 입력 `/reports/{date}` | 404 오류 | “아직 생성되지 않음” 안내 + 생성 CTA | `/reports/{date}` 내 EmptyState + `/reports/new?date={date}` 버튼 |
| F. 생성 중인 active run 날짜 | 토스트 진행만 표시 | 진행 중 안내 유지, 완료 시 결과로 이동 | 기존 `GlobalRunIndicator` 유지 |
| G. API/네트워크 오류 | 404와 동일하게 오류 표시 | 404만 “없음”으로 처리, 500/네트워크는 오류 유지 | 404/기타 오류 분리 |

## 권장 UX 방식
1. **달력 클릭 단계에서 먼저 분기**한다.
   - 날짜가 `statusByDate`에 있으면 `/reports/{date}`로 이동한다.
   - 없으면 `/reports/new?date={date}`로 이동한다.
2. **직접 URL 방어도 추가**한다.
   - `/reports/{date}`에서 API 404이면 “아직 이 날짜 리포트가 없습니다” 상태를 보여준다.
   - CTA: `이 날짜로 뉴스 생성하기`, 보조 CTA: `대시보드로 돌아가기`.
3. **New Report가 query date를 읽는다.**
   - `/reports/new?date=2026-05-10` 진입 시 `runOptions.date`와 Date 입력값을 `2026-05-10`으로 동기화한다.
   - URL date가 잘못된 형식이면 기존 기본 날짜를 유지한다.
