## 개발 목적

현재 자동 파이프라인의 리포트 품질 문제를 해결한다. 특히 섹션별로 **원본 출처 링크와 대표 이미지**가 HTML에 반드시 드러나도록 수집·추출·생성·렌더링 경로를 보강한다.

## 개발 범위

- `Asia/Seoul` 기준 리포트 일자에 맞는 수집 시간창 적용
- RSS/원문 HTML에서 대표 이미지 후보 추출
- `sources` 테이블을 registry 기준으로 동기화하고 `raw_items.source_id` 연결
- 리포트 섹션에 원문 출처 목록과 대표 이미지 URL 저장
- 동일 날짜 리포트 재생성 시 기존 섹션을 안전하게 갱신
- 대표 이미지/출처 표시 흐름에 대한 단위 테스트 추가

## 제외 범위

- 신규 뉴스 소스 대규모 추가
- X/Twitter 유료 API 연동
- 이미지 파일 다운로드·저장·CDN 업로드
- 자동 이미지 생성 모델 호출의 운영 통합
- 클러스터링 알고리즘 전면 재설계

## 잔여 리스크 / 후속 과제

- Anthropic/Meta RSS URL은 현재 응답 실패가 확인되어 registry URL 교체가 필요하다.
- GitHub 수집은 `GITHUB_TOKEN`이 없으면 의도적으로 skip된다.
- 이미지 파일 저장/CDN 업로드와 생성 이미지 fallback은 후속 Phase로 분리한다.

## 리스크 극복 방안

- Anthropic: RSS URL 의존을 제거하고 공식 `https://www.anthropic.com/sitemap.xml` + 뉴스/연구/엔지니어링 URL 패턴 기반 website collector로 대체한다.
- Meta: RSS URL 의존을 제거하고 공식 `https://ai.meta.com/blog/` listing 기반 website collector로 대체한다. `robots.txt`의 sitemap 위치는 기록하되, 실제 접근이 HTML로 반환되는 경우 listing fallback을 사용한다.
- GitHub: 토큰이 없으면 완전 skip하지 않고 낮은 한도 anonymous GitHub API fallback을 사용한다. 토큰이 있으면 기존 한도를 유지한다.
- 이미지: 원본 이미지가 없을 때 텍스트 전용 카드를 만들지 않도록 로컬 SVG fallback 이미지를 자동 생성한다. 원본 이미지가 있으면 원본을 최우선 사용한다.
