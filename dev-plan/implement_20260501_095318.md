## 개발 목적
`Policy & Settings`로 묶여 있던 웹앱 정보구조를 분리해, 편집 정책 관리와 앱/운영 설정의 역할을 명확히 한다. 기존 정책 저장/영구 저장 동작은 유지하면서 메뉴, 라우트, 커맨드 팔레트, 화면 제목이 실제 기능과 일치하도록 정리한다.

## 개발 범위
- `/policy` 라우트 신규 추가 및 현재 `PolicyForm` 이동.
- `/settings` 라우트는 앱/운영 설정 허브 화면으로 전환.
- 사이드바에서 `Policy`와 `Settings`가 각각 독립 경로를 가리키도록 수정.
- 커맨드 팔레트에 `Editorial Policy`와 `Settings` 이동 명령을 분리.
- 관련 UI 테스트를 새 정보구조에 맞게 갱신.

## 제외 범위
- `PolicyForm` 내부 저장 로직, API 계약, YAML persist 동작 변경.
- 신규 백엔드 설정 API 추가.
- 토큰/디자인 시스템 수치 변경.
- 정적 HTML 리포트(`templates/report_newsstream.html.j2`, `public/news/*.html`) 디자인 변경.
- 환경 변수 값을 노출하거나 비밀키 상태를 실제로 조회하는 기능.
