작성 일시: 2026-05-11 19:15 KST

이 문서는 시그널 브리핑형 리포트에서 Refero Design MD Shortlist 기반 visual theme 5개를 웹앱/CLI/API/렌더 경로에 일관되게 적용하기 위한 개발 계획이다.

## 개발 목적
- 사용자가 웹앱에서 `signal_briefing` 출력 형식을 선택했을 때 Refero 기반 visual theme 5개를 선택할 수 있게 한다.
- 기존 `newsstream` 흐름과 현재 Hyperstudio 기반 시그널 브리핑 기본 디자인을 깨지 않는다.
- 미리보기, 파이프라인 실행, CLI 렌더, 저장된 리포트 재렌더가 같은 선택값을 사용하게 한다.

## 개발 범위
- visual theme allowlist:
  - `linear_command_center`
  - `anthropic_research_journal`
  - `cursor_warm_studio`
  - `hyperstudio_terminal_ops`
  - `mercury_twilight_console`
- 기본값: `hyperstudio_terminal_ops`
- 저장 위치: 기존 `Report.method_json.visual_theme` 사용, DB 마이그레이션 없음.
- 적용 템플릿: `templates/report_signal_briefing.html.j2`만 CSS 변수/프리셋으로 분기.
- 웹앱: News Studio Run Options에서 `signal_briefing` 선택 시 visual theme 선택 UI 노출.
- 경로: preview API, run API, CLI `scripts/run_daily.py`, admin render/publish 재렌더에서 값 전달/보존.

## 제외 범위
- `templates/report_newsstream.html.j2` 디자인 변경.
- `docs/design/**` 토큰의 HTML 리포트 자동 적용.
- Refero 외부 사이트 재크롤링.
- 새 출력 스타일 추가.
- DB 스키마 마이그레이션.
