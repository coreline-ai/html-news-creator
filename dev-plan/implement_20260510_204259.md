## 개발 목적

기존 “5분 AI 뉴스 스타일”이라는 외부 사이트 참조 표현을 제품 내부 용어인 **시그널 브리핑형**으로 바꾸고, 웹앱에서 사용자가 기존 **뉴스스트림형**과 신규 **시그널 브리핑형** 중 하나를 선택해 리포트를 생성할 수 있게 한다.

핵심 목표는 디자인 복제가 아니라 출력 구조 선택이다. 신규 시그널 브리핑형은 하루치 AI 이슈를 단일 기사 모음이 아니라 “상단 메타 브리프 → 오늘의 해석 → 흐름별 증거/시사점 → 오늘의 온도/액션”으로 구성한다.

## 용어 정리

| 기존 표현 | 변경 표현 | 의미 |
|---|---|---|
| 5분 AI 뉴스 스타일 | 시그널 브리핑형 | 외부 사이트명이 아닌 내부 제품 출력 모드 |
| 기존 리포트 | 뉴스스트림형 | 현재 `report_newsstream.html.j2` 기반 기본 출력 |
| 트렌드 리포트형 Beta | 시그널 브리핑형 Beta | 데이터/키워드/메인 이벤트/시사점 중심 분석 출력 |

  - `signal_axes`
