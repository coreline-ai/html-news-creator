# implement_20260503_171025.md

작성 일시: `2026-05-03 17:10:25 KST`

이 문서는 GitHub Actions `Daily AI Trend Report` 스케줄 실행 실패의 직접 원인을 수정하고, CI와 Daily workflow의 실행 계약이 다시 벌어지지 않도록 최소 검증을 추가하기 위한 작업 문서다.

## 개발 목적

- Daily workflow가 `scripts/run_daily.py` 실행 시 `app.*` 모듈을 찾지 못해 실패하는 문제를 해결한다.
- README/AGENTS에 명시된 파이프라인 실행 계약(`PYTHONPATH=.`)을 GitHub Actions에도 반영한다.
- dependency 설치 이후 pipeline import를 빠르게 확인해, 장시간 실행 전에 같은 계열 실패를 조기 감지한다.
- 이번 변경은 workflow 실행 경로 수정에 한정하고, 뉴스 수집/생성/렌더링 로직은 변경하지 않는다.

## 개발 범위

- `.github/workflows/daily-report.yml`의 `Run daily pipeline` 실행 명령에 `PYTHONPATH=.` 적용.
- Daily workflow에 pipeline import preflight step 추가.
- workflow YAML 문법/차이 검증.
- 필요한 경우 로컬 unit/ruff smoke 검증.

## 제외 범위

- `scripts/run_daily.py` 파이프라인 로직 변경.
- DB schema, editorial policy, prompt, template, collector/generator 로직 변경.
- GitHub Secrets/Variables 값 설정 자체. `DATABASE_URL`, `OPENAI_BASE_URL`, Netlify/S3 토큰 등은 코드로 해결하지 않고 후속 운영 설정으로 남긴다.
- GitHub-hosted runner를 self-hosted runner로 전환하는 작업.
- 별도 브랜치/PR 생성.

## 잔여 리스크 / 후속 과제

- `DATABASE_URL` secret이 비어 있으면 import 이후 DB 연결 단계에서 실패할 수 있다.
- GitHub-hosted runner에는 로컬 GPT proxy가 없으므로, `OPENAI_BASE_URL`을 접근 가능한 endpoint로 설정하거나 self-hosted runner에서 실행해야 한다.
- `NETLIFY_AUTH_TOKEN`/`NETLIFY_SITE_ID`가 비어 있으면 deploy 단계는 실패할 수 있다. 현재 deploy step은 `continue-on-error: true`라 pipeline 성공 자체를 막지는 않는다.
- `GITHUB_PAT`, S3, Slack secret이 비어 있으면 일부 collector/publish/notify 기능이 제한될 수 있다.

## 최종 결과 요약

- Daily workflow의 직접 실패 원인인 `ModuleNotFoundError: No module named 'app'`는 `PYTHONPATH=.` 적용으로 수정한다.
- 이번 수정은 GitHub Actions workflow만 변경하며, 뉴스 생성 파이프라인 코드에는 영향을 주지 않는다.
